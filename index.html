Privacy Policy — nachosDev Content Uploader

Effective date: 2026-05-22  ·  Last updated: 2026-05-22

1. Who we are

This application ("the App") is a personal content-publishing tool operated by nachosDev (@nachosDev), an independent short-form video creator. The App is not a multi-user product, a website, or a service offered to the public. It is a private command-line utility used solely by its operator to send the operator's own videos to the operator's own TikTok account.

2. What the App does

The App connects to a single TikTok account — the operator's own — through the official TikTok Content Posting API, using the video.upload permission. Its only function is to transfer finished video files from the operator's computer into that same account's TikTok inbox/drafts, where the operator then reviews and publishes them manually inside the TikTok app.

The App does not post publicly on its own, does not operate on any account other than the operator's, and is not available to other users.

3. Information the App accesses

When the operator authorizes the App with TikTok, the App receives and uses the following, strictly to perform the upload:

DataPurposeSource
OAuth access token & refresh tokenAuthenticate API calls on the operator's behalfTikTok OAuth
TikTok user identifier (open_id)Required field for the upload API requestTikTok OAuth
Granted scope (video.upload)Confirm the App has permission to uploadTikTok OAuth
Video filesThe content being uploadedCreated locally by the operator

The App requests no other permissions. It does not read the operator's followers, profile details, analytics, messages, comments, or any other account data. It collects no information from any other person.

4. How information is used

Accessed information is used for one purpose only: to upload the operator's own video files to the operator's own TikTok inbox via the Content Posting API. There is no other processing.

5. How information is stored

  • OAuth tokens are stored only on the operator's local computer, in a single permission-restricted file (.tiktok_token.json). They are never transmitted anywhere except to TikTok's official API endpoints.
  • Video files remain on the operator's local computer.
  • The App runs no server, no database, and no cloud storage. There is no remote system that holds any data.

6. Information sharing

We do not sell, rent, trade, or share any accessed information with any third party. Data flows only between the operator's own computer and TikTok's official API. No advertising networks, analytics providers, or data brokers are involved.

7. Data retention and deletion

  • OAuth tokens persist locally only until they expire or are deleted. The operator can delete them at any time by removing the local token file.
  • The operator can revoke the App's access at any time from TikTok → Settings → Security & permissions → Apps and websites. Revoking access immediately invalidates all stored tokens.
  • Because no data leaves the operator's computer, deleting the local files constitutes complete deletion.

8. Security

OAuth authorization uses the PKCE extension. The token file is created with restricted file permissions. All network communication with TikTok uses HTTPS. The App contains no hardcoded credentials.

9. Children's privacy

The App is a private creator tool and is not directed to children. It does not knowingly collect any information from anyone under 13 (or the applicable minimum age in the operator's jurisdiction).

10. Changes to this policy

If this policy changes, the updated version will be posted at this same URL with a revised "Last updated" date.
